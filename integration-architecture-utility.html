Enterprise Integration Patterns — Utility Context
🗺
Canonical Data Model
«Mediator» · IEC CIM
IEC 61968/61970 CIM as universal data model. All systems speak CIM — adapters transform to/from native formats.
Used: MDMS ↔ SAP ↔ OMS ↔ GIS via Siemens ESB
🔀
Content-Based Router
«Chain of Responsibility»
Routes meter read events based on commodity type (electricity/gas/water) to respective MDMS processing pipeline.
Used: AMI Head-End → multi-commodity MDMS routing
🔄
Message Translator
«Adapter» · GoF
DLMS/COSEM meter data → IEC 61968-9 CIM XML → MDMS canonical format. SOAP IEC 61968 → REST JSON.
Used: All OT protocol → IT system boundaries
🔗
Message Aggregator
«Composite» · GoF
Aggregates 15-min interval reads from 500K meters within collection window before bulk VEE processing in MDMS.
Used: AMI collection window aggregation
💥
Splitter / Scatter-Gather
«Iterator» pattern
Billing run splits by account type, tariff, and territory. Parallel processing across SAP IS-U instances. Gather for consolidated invoice.
Used: SAP billing run orchestration
🔁
Retry / Dead Letter Queue
«Proxy» + «Observer»
Exponential backoff retry for MDMS→SAP IDoc failures. DLQ with alert notification for manual intervention after 3 retries.
Used: All JMS / Kafka error handling
🏪
Message Store / Event Store
«Memento» pattern
Immutable append-only event store for meter reads and grid events. CQRS read model rebuilt from event store for analytics.
Used: MDMS IDR · audit logging
🧱
Circuit Breaker
«Proxy» + resilience
Hystrix/Resilience4j circuit breakers on MDMS→SAP, ESB→OMS calls. Open state triggers fallback to cached grid state.
Used: All synchronous inter-system calls
